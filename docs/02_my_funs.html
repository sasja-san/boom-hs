The Maybe Detour

Haskell has a type called Maybe. It’s one way you could do null values in Haskell, as NULL isn’t exactly a valid value in the language. It’s defined as

Read the declaration as

The datatype Maybe is either Nothing or it’s Just a, were a is some contained data.

data Maybe a = Nothing | Just a

Say that you’re trying to read a number from a String, then Maybe is great way to represent your error:

strToInt :: String -> Maybe Int

If the parsing fails you get Nothing. If the parsing succeeds you get Just x, where x is the number found in the String.

elemIndex :: (Eq a) => a -> [a] -> Maybe Int
elemIndex e xs = ei 0 e xs
  where
    ei :: (Eq a) => Int -> a -> [a] -> Maybe Int
    ei ixCount e    []  = Nothing
    ei ixCount e (x:xs) = if (e == x)
                        then Just ixCount
                        else ei (ixCount + 1) e xs

The Pairing Detour

In Haskell there are “pairs” or “tuples”. They are constructed with the (,) operator/function. So

(,) :: a -> b -> (a,b)

You can always construct a pair with paretheses and a comma.

ghci> let i = 123 :: Int
ghci> let p = (i, "hello")
ghci> :t p
p :: (Int, String)
ghci> p
(123,"hello")

The “tuple” isn’t limited to two elemnts. But when you get up in numbers you really should consider using the data keyword so that your paired up data can have names for each of the fields.

But here it is:

makeTriple :: a -> b -> c -> (a,b,c)
makeTriple x y z = (x,y,z)

For un-pairing you use the functions fst and snd. Write your own versions of fst and snd.

  • takeWhile
  • source
ghci> takeWhile (<3) [1..10]
[1,2]
takeWhile               :: (a -> Bool) -> [a] -> [a]
takeWhile p []          =  []
takeWhile p (x:xs) 
            | p x       =  x : takeWhile p xs
            | otherwise =  []

Using | is called a “guard”. It’s a fancier way of doing multiple nested if-else expressions. otherwise is an alias for True.

  • dropWhile
  • source
ghci> dropWhile (<3) [1..10]
[3,4,5,6,7,8,9,10]
dropWhile               :: (a -> Bool) -> [a] -> [a]
dropWhile p []          =  []
dropWhile p xs@(x:xs')
            | p x       =  dropWhile p xs'
            | otherwise =  xs

The syntax with xs@(x:xs') says that you can deal with the whole list by using xs but you have also named the first element as x and the tail of the list as xs'.

Identity Elements of Operations

Consider multiplication or addition. Both operations have an “identity element” i such that it will return the original value when applying the operation:

x + i_add = x
x * i_mul = x

For addition we have 0 as i_add. For multiplication i_mul is 1.

Also write a function which replaces the element at a certain index. It’s usually looks like this:

(!!=) :: [a] -> (Int, a) -> [a]

Because Haskell lists are singly linked the access time for this is not very nice, so be careful if you actually use this function.
